# 购物车持久化功能实现说明

## 📋 功能概述
实现购物车数据持久化到数据库，用户退出登录后购物车数据不丢失，下次登录时自动恢复。

## 🗄️ 数据库表结构

### cart 表
```sql
CREATE TABLE IF NOT EXISTS `cart` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `user_id` INT NOT NULL,
  `product_id` INT NOT NULL,
  `quantity` INT DEFAULT 1 COMMENT '数量',
  `create_time` DATETIME DEFAULT NOW(),
  `update_time` DATETIME DEFAULT NOW() ON UPDATE NOW(),
  FOREIGN KEY (`user_id`) REFERENCES `user`(`id`),
  FOREIGN KEY (`product_id`) REFERENCES `product`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='购物车表';
```

## 📁 新增文件

1. **CartDao.java** - 购物车DAO接口
   - `saveCartItem()` - 保存商品到购物车
   - `updateCartItemQuantity()` - 更新商品数量
   - `deleteCartItem()` - 删除商品
   - `clearUserCart()` - 清空用户购物车
   - `selectUserCart()` - 查询用户购物车

2. **CartDaoImpl.java** - 购物车DAO实现
   - 实现所有CartDao接口方法
   - 使用DBUtil进行数据库操作
   - 通过LEFT JOIN product表获取商品详细信息

3. **create_cart_table.sql** - 建表SQL脚本
   - 位置：`src/main/resources/create_cart_table.sql`

## 🔧 修改文件

### 1. CartAddServlet.java
**修改内容**：
- 添加CartDao依赖
- 加购商品时同步到数据库
- 区分新增和更新操作

```java
// 同步到数据库（如果用户已登录）
if (user != null) {
    if (exist) {
        cartDao.updateCartItemQuantity(user.getId(), id, item.getQuantity());
    } else {
        cartDao.saveCartItem(user.getId(), id, 1);
    }

```

### 2. CartUpdateServlet.java
**修改内容**：
- 添加CartDao依赖

```java
// 增加数量
if ("inc".equals(op)) {
    item.setQuantity(item.getQuantity() + 1);
    if (user != null) {
```

### 3. LoginServlet.java
**修改内容**：
- 登录成功后从数据库加载购物车
- 将数据库购物车设置到session


## 🔄 完整流程

### 用户加购流程
1. 用户点击"加购"按钮 → CartAddServlet
2. 商品添加到session购物车
3. **同步到数据库**（如果用户已登录）
4. 跳转到购物车页面

### 用户修改购物车流程
1. 用户点击+/-按钮 → CartUpdateServlet
2. 修改session购物车中的数量
3. **同步到数据库**（增加/减少/删除）
4. 刷新购物车页面

### 用户登录流程
1. 用户输入账号密码 → LoginServlet
2. 验证成功，设置session
3. **从数据库加载购物车到session**
4. 跳转到首页

### 用户退出登录流程
1. 用户点击"退出登录" → UserLogoutServlet
2. 清除session（包括loginUser和cart）
3. **数据库购物车保留**
4. 跳转到登录页

### 用户再次登录流程
1. 用户重新登录 → LoginServlet
2. **从数据库恢复购物车到session**
3. 显示上次加购的商品

### 用户结算流程
1. 用户点击"结算" → OrderSubmitServlet
2. 创建订单
3. **清空session购物车**
4. **清空数据库购物车**
5. 跳转到支付页面

## ⚠️ 注意事项

1. **未登录状态**：未登录时购物车只存在于session中，退出后丢失
2. **登录状态**：登录后购物车会同步到数据库，退出后保留
3. **合并策略**：当前实现是登录时直接用数据库购物车覆盖session购物车
4. **结算后清空**：订单创建成功后，购物车（session+数据库）都会被清空

## 🚀 可选优化

1. **购物车合并**：未登录时加购，登录后合并session和数据库购物车
2. **过期清理**：定期清理长时间未登录用户的购物车数据
3. **数量限制**：限制单个用户购物车商品总数
4. **库存校验**：加购时检查商品库存
